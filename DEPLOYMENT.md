# Deployment Guide - Hệ thống Theo dõi Chuyển dạ

## 🚀 Quick Start

### Windows Users

**Option 1: Double-click method (Easiest)**
1. Double-click `deploy.bat`
2. Choose deployment mode (1 for Development, 2 for Production)
3. Follow on-screen instructions

**Option 2: PowerShell**
```powershell
# Development
.\deploy.ps1 development

# Production
.\deploy.ps1 production
```

### Linux/Mac Users

```bash
# Make script executable
chmod +x deploy.sh

# Development
./deploy.sh development

# Production
./deploy.sh production
```

## 📋 What the Script Does

### Automatic Checks
- ✅ Verifies Python 3.8+ is installed
- ✅ Verifies pip is available
- ✅ Checks system compatibility

### Backend Setup
- ✅ Creates Python virtual environment
- ✅ Installs all dependencies from requirements.txt
- ✅ Creates `.env` file with default configuration
- ✅ Initializes database migrations
- ✅ Applies database migrations
- ✅ (Optional) Seeds database with sample data

### Frontend Setup
- ✅ Verifies frontend files exist
- ✅ No build process required (vanilla JS)

### Production Setup (Production mode only)
- ✅ Installs production WSGI server
  - **Linux/Mac**: Gunicorn
  - **Windows**: Waitress
- ✅ Creates production configuration
- ✅ Provides service installation instructions

## 🎯 Deployment Modes

### Development Mode
**Use when:** Testing, developing new features, debugging

**Features:**
- Debug mode enabled
- Auto-reload on code changes
- Detailed error messages
- Sample data seeding option
- Option to auto-start servers

**Servers:**
- Backend: Flask development server (port 5000)
- Frontend: Python HTTP server (port 3000)

### Production Mode
**Use when:** Deploying to production server

**Features:**
- Debug mode disabled
- Optimized for performance
- Production-grade WSGI server
- Service installation support
- Security hardened

**Servers:**
- Backend: Gunicorn/Waitress (port 5000)
- Frontend: Nginx/IIS (recommended)

## 📝 Step-by-Step Instructions

### First Time Deployment

1. **Clone/Download the project**
   ```bash
   git clone <repository-url>
   cd hungvuong
   ```

2. **Run deployment script**
   
   Windows:
   ```powershell
   .\deploy.bat
   ```
   
   Linux/Mac:
   ```bash
   chmod +x deploy.sh
   ./deploy.sh development
   ```

3. **Answer prompts**
   - Seed database? (y/n) → Type `y` for sample data
   - Start servers now? (y/n) → Type `y` to auto-start

4. **Access the application**
   - Frontend: http://localhost:3000
   - Backend: http://localhost:5000

### Updating Deployment

If you already deployed and need to update:

```bash
# Pull latest changes
git pull

# Re-run deployment script
./deploy.sh development  # or deploy.bat on Windows
```

The script will:
- Update dependencies
- Apply new database migrations
- Keep your existing data

## 🔧 Configuration

### Environment Variables (.env)

The script creates a `.env` file automatically. You can customize it:

```bash
# Backend deployment creates: backend/.env
FLASK_ENV=development
FLASK_DEBUG=1
DATABASE_URL=sqlite:///partogram.db
SECRET_KEY=<auto-generated>
CORS_ORIGINS=http://localhost:3000
```

**Important variables:**
- `FLASK_ENV`: development or production
- `FLASK_DEBUG`: 1 for debug mode, 0 for production
- `DATABASE_URL`: Database connection string
- `SECRET_KEY`: Auto-generated secure key
- `CORS_ORIGINS`: Allowed frontend origins

### Port Configuration

Default ports:
- Backend: 5000
- Frontend: 3000

To change ports, edit the script variables:

**deploy.sh (Linux/Mac):**
```bash
BACKEND_PORT=5000
FRONTEND_PORT=3000
```

**deploy.ps1 (Windows):**
```powershell
$BackendPort = 5000
$FrontendPort = 3000
```

## 🐛 Troubleshooting

### Script fails with "Python not found"

**Solution:**
1. Install Python 3.8 or higher from python.org
2. Make sure Python is in PATH
3. Verify: `python --version`

### Script fails with "Permission denied" (Linux/Mac)

**Solution:**
```bash
chmod +x deploy.sh
./deploy.sh development
```

### PowerShell execution policy error (Windows)

**Solution:**
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
.\deploy.ps1 development
```

### Database migration fails

**Solution:**
```bash
# Delete migrations folder and database
rm -rf backend/migrations backend/instance

# Re-run deployment
./deploy.sh development
```

### Port already in use

**Solution:**
```bash
# Find process using the port (Windows)
netstat -ano | findstr :5000

# Kill process
taskkill /PID <process_id> /F

# Or change port in deployment script
```

### CORS errors after deployment

**Solution:**
1. Make sure you access frontend via http://localhost:3000 (not file://)
2. Check `CORS_ORIGINS` in backend/.env matches your frontend URL
3. Restart backend server

## 🔒 Production Deployment

### Linux/Mac Production Setup

1. **Run deployment script**
   ```bash
   ./deploy.sh production
   ```

2. **Install as systemd service** (optional)
   ```bash
   # Script creates hungvuong-partogram.service
   sudo cp hungvuong-partogram.service /etc/systemd/system/
   sudo systemctl daemon-reload
   sudo systemctl enable hungvuong-partogram
   sudo systemctl start hungvuong-partogram
   ```

3. **Setup Nginx**
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       
       location / {
           root /path/to/hungvuong/frontend;
           try_files $uri $uri/ =404;
       }
       
       location /api {
           proxy_pass http://127.0.0.1:5000;
           proxy_set_header Host $host;
       }
   }
   ```

### Windows Production Setup

1. **Run deployment script**
   ```powershell
   .\deploy.ps1 production
   ```

2. **Install as Windows Service** (with NSSM)
   ```powershell
   # Download NSSM from https://nssm.cc/download
   nssm install HungVuongPartogram "C:\path\to\backend\venv\Scripts\python.exe" "C:\path\to\backend\run_production.py"
   nssm start HungVuongPartogram
   ```

3. **Setup IIS** (for frontend)
   - Install IIS
   - Create new site pointing to frontend folder
   - Configure reverse proxy to backend

## 🔄 Starting/Stopping Services

### Development Mode

**Manual start:**
```bash
# Terminal 1: Backend
cd backend
source venv/bin/activate  # Linux/Mac
venv\Scripts\Activate.ps1 # Windows
python app.py

# Terminal 2: Frontend
cd frontend
python -m http.server 3000
```

**Stop:** Press Ctrl+C in each terminal

### Production Mode (systemd - Linux)

```bash
# Start
sudo systemctl start hungvuong-partogram

# Stop
sudo systemctl stop hungvuong-partogram

# Restart
sudo systemctl restart hungvuong-partogram

# Status
sudo systemctl status hungvuong-partogram
```

### Production Mode (NSSM - Windows)

```powershell
# Start
nssm start HungVuongPartogram

# Stop
nssm stop HungVuongPartogram

# Restart
nssm restart HungVuongPartogram
```
